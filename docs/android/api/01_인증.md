## 코드값 정의

### 로그인 제공자 (loginProvider)

사용자가 어떤 방식으로 가입했는지 구분합니다.

| loginProvider | 설명 | 용도 |
|---------------|------|------|
| `KAKAO` | 카카오 로그인 | 카카오 SDK를 통한 소셜 로그인 |
| `EMAIL` | 이메일 로그인 | 이메일/비밀번호 직접 가입 |

### 기기 유형 (deviceType)

FCM 푸시 알림 토큰 등록 시 기기 유형을 구분합니다.

| deviceType | 설명 | 용도 |
|------------|------|------|
| `ANDROID` | 안드로이드 기기 | Android 앱에서 사용 |
| `IOS` | iOS 기기 | iOS 앱에서 사용 (현재 미지원) |

### 비밀번호 규칙

| 항목 | 값 |
|------|-----|
| 전송 방식 | 평문 전송 (HTTPS 필수, 서버에서 BCrypt 해시) |
| 최소 길이 | 8자 |
| 최대 길이 | 72자 (BCrypt 제한) |
| 필수 조건 | 영문 + 숫자 + 특수문자 조합 |

### 닉네임 규칙

| 항목 | 값 |
|------|-----|
| 최소 길이 | 2자 |
| 최대 길이 | 20자 |
| 허용 문자 | 한글, 영문, 숫자 |

### 에러 코드

| 코드 | HTTP 상태 | 메시지 | 설명 | 대응 방법 |
|------|----------|--------|------|----------|
| `AUTH001` | 401 | 카카오 인증 실패 | 유효하지 않은 카카오 access_token | 카카오 SDK로 토큰 재발급 후 재시도 |
| `AUTH002` | 401 | 토큰이 만료되었습니다 | JWT access_token 만료 | refresh_token으로 재발급 |
| `AUTH003` | 401 | 유효하지 않은 토큰입니다 | JWT 형식 오류 또는 변조 | 재로그인 필요 |
| `AUTH004` | 403 | 비활성화된 계정입니다 | 탈퇴 또는 정지된 계정 | 고객센터 문의 |
| `AUTH005` | 400 | 이미 가입된 이메일입니다 | 중복 이메일 회원가입 시도 | 로그인 또는 비밀번호 재설정 안내 |
| `AUTH006` | 400 | 인증 코드가 일치하지 않습니다 | 잘못된 이메일 인증 코드 | 올바른 코드 입력 또는 재발송 |
| `AUTH007` | 400 | 인증 코드가 만료되었습니다 | 10분 경과 | 인증 코드 재발송 요청 |
| `AUTH008` | 400 | 비밀번호가 일치하지 않습니다 | password != passwordConfirm | 비밀번호 확인 재입력 |
| `AUTH009` | 401 | 이메일 또는 비밀번호가 올바르지 않습니다 | 로그인 실패 | 입력 정보 확인 |

## 중복 체크 API

이메일 중복 체크 (회원가입 전 사용)

#### Request

| 파라미터 | 타입 | 필수 | 위치 | 설명 |
|----------|------|------|------|------|
| email | String | Y | Query | 확인할 이메일 |

```
GET /api/v1/auth/check/email?email=user@example.com
```

#### Response

```json
{
  "success": true,
  "message": "Success",
  "code": "OK",
  "data": {
    "exists": true,
    "available": false
  },
  "timestamp": "2025-03-10T14:30:00"
}
```

| 필드 | 타입 | 설명 |
|------|------|------|
| exists | Boolean | 이미 사용 중이면 true |
| available | Boolean | 사용 가능하면 true |

## 1. POST /api/v1/auth/oauth/kakao

카카오 SDK에서 받은 access_token으로 로그인/자동 회원가입

### Request

| 파라미터 | 타입 | 필수 | 위치 | 설명 |
|----------|------|------|------|------|
| accessToken | String | Y | Body | 카카오 SDK Access Token |

```json
{
  "accessToken": "카카오_SDK_ACCESS_TOKEN"
}
```

### Response

```
HTTP 201 Created (신규 회원) / HTTP 200 OK (기존 회원)
```

```json
{
  "success": true,
  "message": "로그인 성공",
  "code": "OK",
  "data": {
    "accessToken": "<access-token>",
    "refreshToken": "<refresh-token>",
    "expiresIn": 21600,
    "isNewUser": true,
    "user": {
      "id": 1,
      "email": "user@example.com",
      "nickname": "홍길동",
      "profileImage": "https://k.kakaocdn.net/...",
      "loginProvider": "KAKAO"
    }
  },
  "timestamp": "2025-03-10T14:30:00"
}
```

### 필드 설명

| 필드 | 타입 | 설명 |
|------|------|------|
| accessToken | String | JWT Access Token (6시간 유효) |
| refreshToken | String | JWT Refresh Token (7일 유효) |
| expiresIn | Long | Access Token 만료 시간 (초 단위, 21600 = 6시간) |
| isNewUser | Boolean | 신규 회원 여부 (자동 가입 시 true) |
| user.id | Long | 사용자 고유 ID |
| user.email | String | 이메일 |
| user.nickname | String | 닉네임 |
| user.profileImage | String? | 프로필 이미지 URL (nullable) |
| user.loginProvider | String | 로그인 방식 ("KAKAO") |

## 이메일 회원가입 플로우

```
[1단계] 이메일 입력 → 인증 메일 발송
    ↓
[2단계] 인증 코드 입력 → 이메일 인증 완료
    ↓
[3단계] 비밀번호/닉네임 입력 → 회원가입 완료 + 로그인
```

## 2. POST /api/v1/auth/signup

이메일 회원가입 1단계: 인증 이메일 발송

### Request

| 파라미터 | 타입 | 필수 | 위치 | 제약조건 | 설명 |
|----------|------|------|------|----------|------|
| email | String | Y | Body | 이메일 형식, 최대 100자 | 이메일 주소 |

```json
{
  "email": "user@example.com"
}
```

### Response

```
HTTP 200 OK
```

```json
{
  "success": true,
  "message": "인증 이메일이 발송되었습니다. 이메일을 확인해주세요.",
  "code": "OK",
  "data": {
    "email": "user@example.com"
  },
  "timestamp": "2025-03-10T14:30:00"
}
```

### 로직

1. 이메일 중복 확인 → 중복 시 AUTH005 에러
2. 6자리 인증 코드 생성 (숫자)
3. DB에 인증 토큰 저장 (10분 유효)
4. 이메일 발송 (Thymeleaf 템플릿)

### 이메일 내용

- 제목: "[WYE] 이메일 인증을 완료해주세요"
- 본문: 6자리 인증 코드 포함
- 유효 시간: 10분

## 4. POST /api/v1/auth/signup/complete

이메일 회원가입 3단계: 비밀번호/닉네임 입력하여 가입 완료

### Request

| 파라미터 | 타입 | 필수 | 위치 | 제약조건 | 설명 |
|----------|------|------|------|----------|------|
| email | String | Y | Body | 이메일 형식 | 인증 완료된 이메일 |
| password | String | Y | Body | 8~72자, 영문+숫자+특수문자 | 비밀번호 |
| passwordConfirm | String | Y | Body | password와 동일 | 비밀번호 확인 |
| nickname | String | Y | Body | 2~20자, 한글/영문/숫자 | 닉네임 |

```json
{
  "email": "user@example.com",
  "password": "Password1!",
  "passwordConfirm": "Password1!",
  "nickname": "투자왕"
}
```

### Response

```
HTTP 201 Created
```

```json
{
  "success": true,
  "message": "회원가입이 완료되었습니다.",
  "code": "OK",
  "data": {
    "accessToken": "<access-token>",
    "refreshToken": "<refresh-token>",
    "expiresIn": 21600,
    "isNewUser": true,
    "user": {
      "id": 5,
      "email": "user@example.com",
      "nickname": "투자왕",
      "profileImage": null,
      "loginProvider": "EMAIL"
    }
  },
  "timestamp": "2025-03-10T14:30:00"
}
```

### 에러 케이스

| 상황 | 에러 코드 |
|------|----------|
| 이메일 인증 미완료 | AUTH006 (EMAIL_NOT_VERIFIED) |
| 비밀번호 불일치 | AUTH007 (PASSWORD_MISMATCH) |
| 이메일 중복 (동시 가입) | USER002 (DUPLICATE_EMAIL) |
| 닉네임 중복 | USER003 (DUPLICATE_NICKNAME) |

## 8. POST /api/v1/auth/token/refresh

Access Token 갱신

### Request

| 파라미터 | 타입 | 필수 | 위치 | 설명 |
|----------|------|------|------|------|
| refreshToken | String | Y | Body | 기존에 발급받은 Refresh Token |

```json
{
  "refreshToken": "<refresh-token>"
}
```

### Response

```
HTTP 200 OK
```

```json
{
  "success": true,
  "message": "토큰 갱신 성공",
  "code": "OK",
  "data": {
    "accessToken": "<access-token>",
    "expiresIn": 21600
  },
  "timestamp": "2025-03-10T14:30:00"
}
```

### 참고

- `refreshToken`, `isNewUser`, `user` 필드는 포함되지 않음
- Refresh Token은 갱신되지 않음 (기존 것 계속 사용)
- Refresh Token 만료 시 재로그인 필요

### 에러 케이스

| 상황 | 에러 코드 |
|------|----------|
| Refresh Token 만료 | AUTH002 |
| 유효하지 않은 토큰 | AUTH003 |

## 9. POST /api/v1/auth/logout

로그아웃 (Refresh Token 폐기, FCM 토큰 삭제)

### Request

**헤더:**
```
Authorization: Bearer {accessToken}
```

| 파라미터 | 타입 | 필수 | 위치 | 설명 |
|----------|------|------|------|------|
| fcmToken | String | N | Body | 현재 기기의 FCM 토큰 (전달 시 해당 기기 푸시 알림 해제) |

```json
{
  "fcmToken": "FCM_DEVICE_TOKEN_STRING"
}
```

> **참고:** Body가 없어도 정상 동작합니다. FCM 토큰을 전달하면 해당 기기의 푸시 알림만 해제되고, 다른 기기는 계속 푸시를 받습니다.

### Response

```
HTTP 200 OK
```

```json
{
  "success": true,
  "message": "로그아웃되었습니다.",
  "code": "OK",
  "timestamp": "2025-03-10T14:30:00"
}
```

## 13. POST /api/v1/auth/fcm/token

FCM 토큰 등록 (푸시 알림용)

**인증 필요:** `Authorization: Bearer {accessToken}`

### Request

| 파라미터 | 타입 | 필수 | 위치 | 제약조건 | 설명 |
|----------|------|------|------|----------|------|
| token | String | Y | Body | 최대 500자 | FCM 디바이스 토큰 |
| deviceType | String | Y | Body | ANDROID / IOS | 기기 유형 |

```json
{
  "token": "FCM_DEVICE_TOKEN_STRING",
  "deviceType": "ANDROID"
}
```

### Response

```
HTTP 201 Created
```

```json
{
  "success": true,
  "message": "FCM 토큰이 등록되었습니다.",
  "code": "OK",
  "timestamp": "2025-03-10T14:30:00"
}
```

### 참고

- 로그인 직후 호출 권장
- 동일 사용자가 여러 기기 사용 시 각 기기별 등록
- **로그아웃 시 자동 삭제** (별도 DELETE 불필요)
