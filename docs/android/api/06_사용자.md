# 사용자 API

> 화면: MyPageScreen.kt, FavoriteListScreen.kt

---

## 코드값 정의

### 사용자 역할 (role)

| role | 설명 | 권한 |
|------|------|------|
| `USER` | 일반 사용자 | 기본 기능 사용 |
| `ADMIN` | 관리자 | 관리 기능 포함 |

### 에러 코드

| 코드 | HTTP 상태 | 메시지 | 설명 | 대응 방법 |
|------|----------|--------|------|----------|
| `USER001` | 404 | 사용자를 찾을 수 없습니다 | 존재하지 않는 userId | - |
| `USER002` | 409 | 이미 사용 중인 닉네임입니다 | 닉네임 중복 | 다른 닉네임 사용 |
| `USER003` | 404 | ETF를 찾을 수 없습니다 | 존재하지 않는 ticker | ETF 목록 새로고침 |
| `USER004` | 409 | 이미 관심 등록된 ETF입니다 | 중복 관심 등록 | - |
| `USER005` | 404 | 관심 등록되지 않은 ETF입니다 | 관심 삭제 시 해당 ETF 없음 | - |
| `FILE001` | 500 | 파일 업로드에 실패했습니다 | 서버 저장 실패 | 재시도 |
| `FILE002` | 400 | 파일이 비어있습니다 | 파일 미선택 | 파일 선택 후 재시도 |
| `FILE003` | 400 | 파일 크기가 제한을 초과했습니다 (최대 5MB) | 파일 용량 초과 | 5MB 이하 파일 사용 |
| `FILE004` | 400 | 허용되지 않는 파일 형식입니다 | 지원하지 않는 확장자 | jpg, jpeg, png, gif, webp 사용 |

## 1. GET /api/v1/users/me

내 정보 조회

**인증 필요:** `Authorization: Bearer {accessToken}`

### Request

파라미터 없음

### Response

```
HTTP 200 OK
```

```json
{
  "success": true,
  "message": "Success",
  "code": "OK",
  "data": {
    "id": 1,
    "email": "user@example.com",
    "nickname": "투자왕",
    "profileImage": "https://...",
    "role": "USER",
    "isActive": true,
    "lastLoginAt": "2025-03-10T14:30:00",
    "createdAt": "2025-01-15T10:00:00",
    "socialAccounts": [
      {
        "provider": "KAKAO",
        "email": "user@example.com",
        "isPrimary": true,
        "linkedAt": "2025-01-15T10:00:00"
      }
    ]
  },
  "timestamp": "2025-03-10T14:30:00"
}
```

### 필드 설명

| 필드 | 타입 | 설명 |
|------|------|------|
| id | Long | 사용자 고유 ID |
| email | String | 이메일 |
| nickname | String | 닉네임 |
| profileImage | String? | 프로필 이미지 URL (nullable) |
| role | String | 사용자 역할 (USER / ADMIN) |
| isActive | Boolean | 계정 활성화 상태 |
| lastLoginAt | String | 마지막 로그인 시각 |
| createdAt | String | 가입일 |
| socialAccounts | Array | 연결된 소셜 계정 목록 |

## 4. DELETE /api/v1/users/me

회원 탈퇴

**인증 필요:** `Authorization: Bearer {accessToken}`

### Request

바디 없음

### Response

```
HTTP 200 OK
```

```json
{
  "success": true,
  "message": "회원 탈퇴 완료",
  "code": "OK",
  "timestamp": "2025-03-10T14:30:00"
}
```

### 로직

1. 프로필 이미지 삭제
2. 관련 데이터 삭제:
   - 알림, 알림 설정, FCM 토큰
   - 로그인 이력, 비밀번호 재설정 토큰, 리프레시 토큰
   - 포트폴리오 AI 피드백, 시뮬레이션
   - 포트폴리오, 관심 ETF, 보유 ETF
3. 소셜 계정 연결 해제 (cascade 삭제)
4. 사용자 영구 삭제 (hard delete)

**주의:** 삭제된 데이터는 복구할 수 없습니다.

## 5. POST /api/v1/users/me/profile-image

프로필 이미지 업로드

**인증 필요:** `Authorization: Bearer {accessToken}`

### Request

| 파라미터 | 타입 | 필수 | 위치 | 제약조건 | 설명 |
|----------|------|------|------|----------|------|
| file | MultipartFile | Y | Form-Data | 최대 5MB, jpg/jpeg/png/gif/webp | 이미지 파일 |

**Content-Type:** `multipart/form-data`

```
POST /api/v1/users/me/profile-image
Content-Type: multipart/form-data

file: [이미지 파일 바이너리]
```

### 파일 제약조건

| 항목 | 제약 |
|------|------|
| 최대 크기 | 5MB |
| 허용 확장자 | jpg, jpeg, png, gif, webp |
| 저장 경로 | /uploads/profiles/ |
| 파일명 | UUID 자동 생성 (예: a1b2c3d4-5678-90ab-cdef.jpg) |

### Response

```
HTTP 201 Created
```

```json
{
  "success": true,
  "message": "프로필 이미지 업로드 완료",
  "code": "OK",
  "data": "https://api.example.com/uploads/profiles/a1b2c3d4-5678-90ab-cdef.jpg",
  "timestamp": "2025-03-10T14:30:00"
}
```

### 로직

1. 파일 유효성 검사 (크기, 확장자)
2. 기존 프로필 이미지 존재 시 삭제
3. UUID 기반 고유 파일명 생성
4. 파일 저장 후 URL 반환
5. 사용자 profileImage 필드 업데이트

### 에러 케이스

| 상황 | 에러 코드 |
|------|----------|
| 파일 미선택 | FILE002 |
| 5MB 초과 | FILE003 |
| 지원하지 않는 확장자 | FILE004 |
| 저장 실패 | FILE001 |

## 7. GET /api/v1/users/me/favorites/etfs

관심 ETF 목록 조회

**인증 필요:** `Authorization: Bearer {accessToken}`

### Request

| 파라미터 | 타입 | 필수 | 위치 | 기본값 | 설명 |
|----------|------|------|------|--------|------|
| sort | String | N | Query | RECENT | 정렬 기준 |

### 정렬 옵션 (sort)

| 값 | 설명 |
|----|------|
| `RECENT` | 최근 등록순 (기본값) |
| `CHANGE_RATE_DESC` | 등락률 높은순 |
| `CHANGE_RATE_ASC` | 등락률 낮은순 |
| `NAME_ASC` | 이름순 (가나다) |

**요청 예시:**
```
GET /api/v1/users/me/favorites/etfs?sort=CHANGE_RATE_DESC
```

### Response

```
HTTP 200 OK
```

```json
{
  "success": true,
  "message": "Success",
  "code": "OK",
  "data": {
    "favorites": [
      {
        "ticker": "091160",
        "name": "KODEX 반도체",
        "riskType": "위험중립형",
        "assetManager": "삼성자산운용",
        "currentPrice": 45230,
        "changeRate": 1.24,
        "favoritedAt": "2025-03-01T10:00:00"
      },
      {
        "ticker": "091180",
        "name": "TIGER 반도체",
        "riskType": "적극투자형",
        "assetManager": "미래에셋자산운용",
        "currentPrice": 32150,
        "changeRate": -0.55,
        "favoritedAt": "2025-03-05T14:30:00"
      }
    ],
    "totalCount": 2
  },
  "timestamp": "2025-03-10T14:30:00"
}
```

### 필드 설명

| 필드 | 타입 | 설명 |
|------|------|------|
| ticker | String | ETF 종목코드 (6자리) |
| name | String | ETF 명칭 |
| riskType | String | 위험 유형 (안정형, 안정추구형, 위험중립형, 적극투자형, 공격투자형) |
| assetManager | String | 자산운용사 |
| currentPrice | BigDecimal | 현재가 (최신 종가) |
| changeRate | BigDecimal | 등락률 (%) |
| favoritedAt | String | 관심 등록일 |
| totalCount | Int | 관심 ETF 총 개수 |
