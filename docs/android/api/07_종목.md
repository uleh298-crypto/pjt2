## GET /api/v1/stocks/{ticker}/tags

종목의 태그(시장유형, 산업그룹, 세분류)를 조회합니다.

### Request

| 파라미터 | 타입 | 필수 | 위치 | 설명 |
|----------|------|------|------|------|
| ticker | String | Y | Path | 종목 티커 (6자리) |

```
GET /api/v1/stocks/005930/tags
```

### Response

```
HTTP 200 OK
```

```json
{
  "success": true,
  "message": "Success",
  "code": "OK",
  "data": [
    "KOSPI",
    "반도체",
    "메모리 반도체 (DRAM/NAND)"
  ],
  "timestamp": "2026-03-17T10:24:44.2420295"
}
```

### 필드 설명

| 필드 | 타입 | 설명 |
|------|------|------|
| data | Array<String> | 태그 목록 [시장유형, 그룹명, 세분류명] |

### 태그 구성

| 순서 | 출처 | 예시 |
|------|------|------|
| 1 | stock.market_type | KOSPI, KOSDAQ |
| 2 | industry_classification.group_name | 반도체, 소프트웨어, 전자/IT |
| 3 | industry_classification.name | 메모리 반도체, 플랫폼/커머스 |

## GET /api/v1/stocks/{ticker}/related

같은 산업 그룹의 관련 종목 3개를 조회합니다.

### Request

| 파라미터 | 타입 | 필수 | 위치 | 설명 |
|----------|------|------|------|------|
| ticker | String | Y | Path | 종목 티커 (6자리) |

```
GET /api/v1/stocks/005930/related
```

### Response

```
HTTP 200 OK
```

```json
{
  "success": true,
  "message": "Success",
  "code": "OK",

    {
      "ticker": "003160",

    {
      "ticker": "000990",

      "industryName": "시스템 반도체 (비메모리/AP)",
      "relationType": "동종 업계",
      "logoUrl": "https://api.example.com/uploads/logos/000990.png"

    }
  ],
  "timestamp": "2026-03-17T10:28:59.3198763"

```

### 필드 설명

| 필드 | 타입 | 설명 |
|------|------|------|
| ticker | String | 종목 티커 |
| companyName | String | 회사명 |
| industryCode | String | 산업분류 코드 |
| industryName | String | 산업분류명 (세분류) |
| relationType | String | 관계 유형 (동종 업계) |
| logoUrl | String | 회사 로고 URL (`/uploads/logos/{ticker}.png`) |

### 조회 로직

1. 기준 종목의 `industry_code` 조회
2. `industry_classification` 테이블에서 해당 코드의 `group_name` 확인
3. 같은 `group_name`을 가진 다른 종목 조회 (상장주식수 기준 정렬)
4. 자기 자신 제외, 상위 3개 반환
