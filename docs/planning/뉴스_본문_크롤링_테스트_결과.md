## 4. 자동화 시스템 (구현 완료)

> ERD에 news_source 테이블 추가 완료

### 4.1 문제점
- 언론사별 크롤링 가능 여부가 수시로 변경됨
- 가능 → 불가: 봇 차단 정책 강화
- 불가 → 가능: 정책 완화, URL 구조 변경

### 4.2 news_source 테이블

> 위치: docs/sql/ERD.sql, db/init.sql

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | BIGSERIAL | PK |
| name | VARCHAR(50) | 언론사명 |
| domain | VARCHAR(100) | 도메인 (UNIQUE) |
| is_content_available | BOOLEAN | 본문 크롤링 가능 여부 |
| last_success_at | TIMESTAMP | 마지막 성공 시점 |
| last_failure_at | TIMESTAMP | 마지막 실패 시점 |
| failure_count | INTEGER | 연속 실패 횟수 (5회 이상 시 자동 비활성화) |
| css_selector | TEXT | 본문 CSS 선택자 |
| notes | TEXT | 비고 |
| created_at | TIMESTAMP | 생성일시 |
| updated_at | TIMESTAMP | 수정일시 |

### 4.3 자동화 로직

```
1. 뉴스 수집 시 본문 크롤링 시도
2. 실패 시 failure_count 증가
3. failure_count >= 5 → is_content_available = false
4. 주기적으로 (1일 1회) 비활성 언론사 재테스트
5. 재테스트 성공 시 is_content_available = true, 기존 기사 본문 재수집
```

### 4.4 모니터링 (관리자 기능)

- 언론사별 크롤링 성공률 대시보드
- 실패율 급증 시 알림
- 신규 언론사 수동 등록

### 5.2 아키텍처 요약

```
[Naver News Search API 응답]
        │
        ├── link: "https://n.news.naver.com/..."     ← 본문 크롤링용
        │
        └── originallink: "https://mk.co.kr/..."    ← 언론사명 추출용
                │
                ▼
        _extract_source() → "매일경제"


[처리 흐름]

  naver_scraper.py
        │
        ├── source_url = link (네이버 뉴스 URL)
        │         │
        │         ▼
        │   content_scraper.py → 본문 + 썸네일 추출
        │
        └── source = _extract_source(originallink) → 언론사명

        ▼
  NewsArticle (source="매일경제", content="...", thumbnail_url="...")
```
