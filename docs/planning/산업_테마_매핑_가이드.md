# 산업 테마 매핑 가이드

## 개요

WhatsYourETF 앱에서는 **21개 DB 그룹코드**를 **14개 모바일 UI 테마**로 통합하여 사용합니다.

```
┌─────────────────┐      매핑       ┌─────────────────┐
│  21개 group_code │  ──────────▶  │  14개 UI 테마    │
│  (DB 저장값)     │               │  (화면 표시용)   │
└─────────────────┘               └─────────────────┘
```

---

## 기능별 사용 구분

| 기능 | 사용하는 코드 | 설명 |
|-----|-------------|------|
| **ETF 클러스터 (버블 차트)** | 21개 group_code 그대로 | 세밀한 섹터 분포 시각화 |
| **ETF 테마 검색 (목록)** | 14개 UI 테마 | 사용자가 테마 선택 → 21개로 변환하여 쿼리 |
| **뉴스 카테고리 필터** | 13개 뉴스 카테고리 | `NEWS_*` 코드 직접 사용 |
| **뉴스 크롤링** | 21개 → 13개 변환 | 종목의 group_code를 뉴스 카테고리로 변환 |

### ETF 클러스터가 21개를 그대로 사용하는 이유
- 버블 차트에서 **세밀한 섹터 분포**를 보여주기 위함
- 예: "화학/소재" 하나로 묶으면 CHEM(화학)과 STEEL(철강)의 비중 차이를 알 수 없음
- 클러스터 데이터: `etf_sector_cluster.group_code` = 21개 값

## ETF.sector vs group_code (독립적인 분류 체계)

**두 분류 체계는 직접 매핑되지 않습니다. 각각 독립적으로 사용됩니다.**

| 분류 | 대상 | 출처 | 저장 위치 | 용도 |
|------|------|------|----------|------|
| ETF.sector | ETF 자체 | KRX (12개) | `etf.sector` | ETF 목록/상세 화면 표시 |
| group_code | 개별 회사 | 내부 매핑 (21개) | `company_info.industry_group` | 클러스터 버블 차트 |

```
┌─────────────────────┐                    ┌─────────────────────┐
│  etf 테이블          │                    │ company_info 테이블  │
│  sector (KRX 12개)  │                    │ industry_group      │
│  예: "전자 / IT"     │                    │ 예: "IT_SEMI"       │
└─────────────────────┘                    └─────────────────────┘
         │                                          │
         │ ETF 화면 표시용                           │
         ▼                                          ▼
┌─────────────────────┐                    ┌─────────────────────┐
│  ETF 목록/상세 화면  │    etf_stock_      │  etf_sector_cluster │
│  섹터: "전자 / IT"   │◀── composition ──▶│  버블 차트 데이터   │
└─────────────────────┘    (구성종목 연결)   └─────────────────────┘
```

### KRX 12개 섹터 (etf.sector)

ETF 자체에 KRX가 부여한 분류:
- 건설, 금융, 바이오 / 의약, 에너지 / 유틸리티, 운송
- 유통 / 소매, 자동차, 전자 / IT, 철강 / 금속, 통신 / 미디어
- 화학 / 소재, 기타

### ETF 클러스터 생성 과정

ETF의 `sector`와 클러스터의 `group_code`는 **직접 매핑이 아닌 구성종목 집계**로 연결됩니다:

```
1. ETF 구성종목 조회 (etf_stock_composition)
2. 각 종목의 group_code 조회 (company_info.industry_group)
3. 비중(weight_pct) 기준 집계
4. etf_sector_cluster 테이블에 저장
```

**예시: KODEX 반도체 ETF**
- `etf.sector` = "전자 / IT" (KRX 분류)
- 구성종목 집계 결과 (etf_sector_cluster):
  - IT_SEMI: 85%
  - IT_ELEC: 10%
  - IT_SW: 5%

## 데이터 흐름

### 1. ETF 탐색 화면

```
[사용자]                    [프론트엔드]                    [백엔드]
   │                            │                            │
   │  "IT/전자" 테마 선택        │                            │
   │ ─────────────────────────▶ │                            │
   │                            │  GET /api/v1/etf?theme=IT  │
   │                            │ ─────────────────────────▶ │
   │                            │                            │
   │                            │   IndustryGroupMapper      │
   │                            │   .getGroupCodesForTheme   │
   │                            │   ("IT")                   │
   │                            │   → ["IT_ELEC", "IT_SW"]   │
   │                            │                            │
   │                            │   SELECT * FROM etf        │
   │                            │   WHERE group_code IN      │
   │                            │   ('IT_ELEC', 'IT_SW')     │
   │                            │ ◀───────────────────────── │
   │  ETF 목록 표시              │                            │
   │ ◀───────────────────────── │                            │
```

### 2. 뉴스 카테고리 필터링

```
[사용자]                    [프론트엔드]                    [백엔드]
   │                            │                            │
   │  "IT/전자" 카테고리 선택    │                            │
   │ ─────────────────────────▶ │                            │
   │                            │  GET /api/v1/news          │
   │                            │  ?category=NEWS_IT         │
   │                            │ ─────────────────────────▶ │
   │                            │                            │
   │                            │   SELECT * FROM news       │
   │                            │   WHERE category_code      │
   │                            │   = 'NEWS_IT'              │
   │                            │ ◀───────────────────────── │
   │  뉴스 목록 표시             │                            │
   │ ◀───────────────────────── │                            │
```

### 3. 뉴스 크롤링 시 카테고리 할당

```
[크롤러]                                      [DB]
   │                                           │
   │  삼성전자(005930) 관련 뉴스 수집           │
   │  → company_info.industry_group = "IT_SEMI" │
   │                                           │
   │  IndustryGroupMapper                      │
   │  .toNewsCategoryCode("IT_SEMI")           │
   │  → "NEWS_SEMI"                            │
   │                                           │
   │  INSERT INTO news_article                 │
   │  (category_code = 'NEWS_SEMI')            │
   │ ─────────────────────────────────────────▶│
```

## 코드 사용법

### Java (user-service)

```java
import com.whatsyouretf.userservice.domain.common.util.IndustryGroupMapper;
import com.whatsyouretf.userservice.domain.common.util.IndustryGroupMapper.MobileTheme;

// 1. group_code → UI 테마 변환
MobileTheme theme = IndustryGroupMapper.toMobileTheme("IT_ELEC");
// → MobileTheme.IT

String displayName = IndustryGroupMapper.toDisplayName("IT_ELEC");
// → "IT/전자"

// 2. UI 테마 → group_code 목록 (ETF 검색용)
List<String> codes = IndustryGroupMapper.getGroupCodesForTheme("IT");
// → ["IT_ELEC", "IT_SW"]

// 3. group_code → 뉴스 카테고리 코드
String newsCategory = IndustryGroupMapper.toNewsCategoryCode("STEEL");
// → "NEWS_CHEM" (화학/소재로 통합)

// 4. 전체 테마 목록
List<MobileTheme> allThemes = IndustryGroupMapper.getAllThemes();
// → [SEMI, IT, BIO, AUTO, CHEM, ENERGY, FINANCE, ...]

// 5. 유효성 검사
boolean validTheme = IndustryGroupMapper.isValidThemeCode("IT");     // true
boolean validGroup = IndustryGroupMapper.isValidGroupCode("IT_ELEC"); // true
```

### Python (data-service)

```python
# stock_news_scraper.py의 _get_category_from_industry() 메서드 사용
category = scraper._get_category_from_industry("IT_ELEC")
# → "NEWS_IT"
```
