## 코드값 정의

### 뉴스 카테고리 (categoryCode)

뉴스를 투자 테마별로 분류하는 카테고리입니다. ETF 섹터와 연관되어 관련 뉴스 매칭에 사용됩니다.

| categoryCode | 화면 표시명 | 설명 | 관련 ETF 섹터 |
|--------------|------------|------|--------------|
| `NEWS_SEMI` | 반도체 | 반도체 산업 관련 뉴스 | 반도체, 메모리 |
| `NEWS_IT` | IT/전자 | IT, 전자제품, 소프트웨어 뉴스 | IT, 테크, 소프트웨어 |
| `NEWS_BIO` | 바이오/의약 | 제약, 바이오, 헬스케어 뉴스 | 바이오, 헬스케어 |
| `NEWS_AUTO` | 자동차 | 자동차, 전기차, 모빌리티 뉴스 | 자동차, 2차전지 |
| `NEWS_CHEM` | 화학/소재 | 화학, 소재, 철강 뉴스 | 화학, 소재 |
| `NEWS_ENERGY` | 에너지 | 에너지, 신재생에너지, 석유 뉴스 | 에너지, 신재생 |
| `NEWS_FINANCE` | 금융 | 은행, 증권, 보험 뉴스 | 금융, 은행 |
| `NEWS_CONSTRUCT` | 건설/부동산 | 건설, 부동산, 인프라 뉴스 | 건설, 부동산 |
| `NEWS_CONSUMER` | 소비재 | 유통, 식품, 의류 뉴스 | 소비재, 유통 |
| `NEWS_TELECOM` | 통신/미디어 | 통신, 방송, 미디어 뉴스 | 통신, 미디어 |
| `NEWS_TRANSPORT` | 운송/물류 | 항공, 해운, 물류 뉴스 | 운송, 물류 |
| `NEWS_INDUSTRY` | 산업재 | 기계, 조선, 방산 뉴스 | 산업재, 기계 |
| `NEWS_ETC` | 기타 | 기타 산업 뉴스 | 기타 |

### 에러 코드

| 코드 | HTTP 상태 | 메시지 | 설명 | 대응 방법 |
|------|----------|--------|------|----------|
| `NEWS001` | 404 | 뉴스를 찾을 수 없습니다 | 존재하지 않는 newsId | 목록 새로고침 |
| `NEWS002` | 400 | 유효하지 않은 카테고리입니다 | 잘못된 categoryCode | 올바른 카테고리 코드 사용 |

## 1. GET /api/v1/news

뉴스 목록 조회 (커서 기반 페이징, 무한 스크롤)

### Request

| 파라미터 | 타입 | 필수 | 위치 | 기본값 | 설명 |
|----------|------|------|------|--------|------|
| category | String | N | Query | null | 뉴스 카테고리 코드. 미입력 시 전체 카테고리 조회 |
| lastId | Long | N | Query | null | 마지막 조회 뉴스 ID. 첫 페이지는 생략 |
| size | int | N | Query | 20 | 페이지 크기 (최대 50) |

**카테고리 코드:**

상단 "뉴스 카테고리 (categoryCode)" 표 참조

**요청 예시:**
```
GET /api/v1/news                              # 첫 페이지 (20개)
GET /api/v1/news?size=30                      # 첫 페이지 (30개)
GET /api/v1/news?lastId=3080                  # ID 3080 이후 20개
GET /api/v1/news?category=NEWS_SEMI           # 반도체 뉴스 첫 페이지
GET /api/v1/news?category=NEWS_SEMI&lastId=500  # 반도체 뉴스 ID 500 이후
```

### Response

```json
{
  "success": true,
  "message": "Success",
  "code": "OK",
  "data": {
    "news": [
      {
        "id": 123,
        "title": "삼성전자, AI 반도체 대규모 투자 발표",
        "source": "한국경제",
        "thumbnailUrl": "https://...",
        "categoryCode": "NEWS_SEMI",
        "categoryName": "반도체",
        "publishedAt": "2025-03-10T09:30:00"
      }
    ],
    "size": 20,
    "hasMore": true,
    "nextCursor": 103
  },
  "timestamp": "2025-03-10T14:30:00"
}
```

### 화면 매핑

| API 필드 | 화면 표시 |
|----------|----------|
| title | 뉴스 제목 (2줄 max) |
| source | 언론사명 |
| thumbnailUrl | 썸네일 이미지 |
| publishedAt | "3시간 전" 형식으로 변환 |
| categoryName | 카테고리 칩 |

### 페이징 응답 필드

| 필드 | 타입 | 설명 |
|------|------|------|
| size | int | 페이지 크기 |
| hasMore | boolean | 다음 페이지 존재 여부 (무한 스크롤 종료 조건) |
| nextCursor | Long | 다음 페이지 요청 시 사용할 lastId (hasMore=false면 null) |

### 무한 스크롤 구현 가이드

```kotlin
// 1. 첫 페이지 로드
val response = api.getNews()  // 기본 20개

// 2. 스크롤 끝 도달 시 다음 페이지 로드
if (response.hasMore) {
    val nextPage = api.getNews(lastId = response.nextCursor)
}

// 3. hasMore = false면 로딩 종료
```

**커서 기반 페이징 장점:**
- 새 뉴스가 추가되어도 중복/누락 없음
- 무한 스크롤에 최적화

## 3. GET /api/v1/news/search

뉴스 검색 (최신 20개)

### Request

| 파라미터 | 타입 | 필수 | 위치 | 제약조건 | 설명 |
|----------|------|------|------|----------|------|
| keyword | String | Y | Query | 1~50자 | 검색어 (제목 LIKE 검색) |
| category | String | N | Query | - | 카테고리 코드. 지정 시 해당 카테고리 내에서 검색 |

**카테고리 코드:** 상단 "뉴스 카테고리 (categoryCode)" 표 참조

**요청 예시:**
```
GET /api/v1/news/search?keyword=삼성전자
GET /api/v1/news/search?keyword=반도체
GET /api/v1/news/search?keyword=삼성&category=NEWS_SEMI
GET /api/v1/news/search?keyword=투자&category=NEWS_FINANCE
```

### Response

뉴스 목록과 동일한 구조 + `keyword` 필드 추가

```json
{
  "success": true,
  "message": "Success",
  "code": "OK",
  "data": {
    "news": [...],
    "keyword": "삼성전자",
    "size": 20,
    "hasMore": false,
    "nextCursor": null
  },
  "timestamp": "2025-03-10T14:30:00"
}
```

> 검색 결과는 최신 20개만 반환하며 페이징 미지원

## 4. GET /api/v1/news/portfolio/{portfolioId}

포트폴리오 관련 뉴스 조회 (매일 오전 9시 캐시 갱신)

### Request

| 파라미터 | 타입 | 필수 | 위치 | 설명 |
|----------|------|------|------|------|
| portfolioId | Long | Y | Path | 포트폴리오 ID |

**요청 예시:**
```
GET /api/v1/news/portfolio/1
```

### Response

```json
{
  "success": true,
  "message": "Success",
  "code": "OK",
  "data": {
    "portfolioId": 123,
    "portfolioName": "기술주 중심 성장 전략",
    "news": [
      {
        "id": 456,
        "title": "미국 ETF 시장, 인공지능 테마로 자금 유입 역대 최고 기록",
        "summary": "글로벌 AI 경쟁 가속화로 관련 ETF 수익률 고공행진...",
        "source": "금융경제뉴스",
        "thumbnailUrl": "https://...",
        "publishedAt": "2025-03-10T09:30:00"
      }
    ],
    "updatedAt": "2025-03-10T09:00:00"
  },
  "timestamp": "2025-03-10T14:30:00"
}
```

### 로직

1. 포트폴리오 구성 ETF의 모든 구성종목 조회
2. 해당 종목들과 관련된 뉴스 매핑
3. **투자금액 기반 관련성 스코어**로 정렬 후 상위 5개 반환
4. 매일 오전 9시 캐시 갱신

### 관련성 스코어 계산

```
score = etf_count × close × weight_pct × recency_multiplier
recency_multiplier = 1 / (1 + 경과일수 × 0.3)
```

| 항목 | 설명 | 출처 |
|------|------|------|
| `etf_count` | ETF 보유 수량 | portfolio_etf 테이블 |
| `close` | 전일 종가 | etf_prices 테이블 (최신 거래일) |
| `weight_pct` | ETF 내 종목 비중 (%) | etf_stock_composition 테이블 |
| `recency_multiplier` | 최신성 가중치 | 뉴스 발행일 기준 계산 |

**최신성 가중치 예시:**

| 경과일 | recency_multiplier |
|--------|-------------------|
| 0일 | 1.0 |
| 1일 | 0.77 |
| 3일 | 0.53 |
| 7일 | 0.32 |

**스코어 계산 예시:**
- ETF 10주 보유, 전일 종가 50,000원, 종목 비중 5%, 1일 경과
- score = 10 × 50,000 × 5 × 0.77 = 1,925,000

**정렬:**
1. 스코어 높은 순으로 상위 5개 뉴스 선택
2. 선택된 5개를 `published_at DESC`로 최신순 정렬하여 반환

### relatedEtfs 조회 로직

```
news_stock_mapping (뉴스 → 종목)
    → stock.company_id (종목 → 회사)
    → etf_stock_composition (회사 → ETF 구성)

```
